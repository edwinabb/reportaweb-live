### **TICKET: REP-3.11-003**

**TITLE:** Agregar campos progreso_porcentaje + tarea_padre_id en tareas

**TYPE:** Task  
**PRIORITY:** 🔴 CRITICAL

**EPIC:** v3.11 Database Recovery  
**ASSIGNEE:** [Data Engineer]  
**DUE DATE:** 2026-07-18  
**STATUS:** Ready for Backlog

---

#### DESCRIPTION

Bubble tiene 2 campos críticos que **NO existen en Supabase**:

1. **progreso_porcentaje (NUMERIC 5,2)** — 400+ tareas en Bubble tienen datos
   - Usado en Dashboard para mostrar progreso de tareas
   - Usado en Reportes para análisis de avance
   - Sin este campo → Dashboard incompleto

2. **tarea_padre_id (UUID FK tareas.id)** — 50 tareas son "subtareas"
   - Usado en Planificación para expandir subtareas
   - Estructura jerárquica de tareas
   - Sin este campo → No se pueden tener subtareas

3. **estado_seniat (TEXT)** — Compliance/Reporting (bajo prioridad, documentar en v3.12)

**Severidad:** 🔴 CRÍTICA — Dashboard + Planificación incompletos

---

#### ACCEPTANCE CRITERIA

- [ ] Crear columna `progreso_porcentaje NUMERIC(5,2)` en `tareas`
- [ ] Crear columna `tarea_padre_id UUID REFERENCES tareas(id)` en `tareas`
- [ ] Migrar datos de Bubble: 400+ valores de progreso
- [ ] Migrar datos de Bubble: 50 relaciones padre-hijo
- [ ] Validar RLS policy funciona con nueva FK
- [ ] Test: Dashboard muestra progreso_porcentaje
- [ ] Test: Planificación expande subtareas correctamente

---

#### TASKS

1. **Create Migration SQL**
   ```sql
   -- Migration file: supabase/migrations/YYYYMMDDHHMMSS_add_progreso_tarea_padre.sql
   
   ALTER TABLE tareas ADD COLUMN progreso_porcentaje NUMERIC(5,2) DEFAULT 0;
   ALTER TABLE tareas ADD COLUMN tarea_padre_id UUID REFERENCES tareas(id) ON DELETE SET NULL;
   
   -- Index para queries de subtareas
   CREATE INDEX idx_tareas_padre ON tareas(tarea_padre_id);
   ```

2. **Migrate Data from Bubble**
   ```sql
   -- Update progreso_porcentaje
   UPDATE tareas t 
   SET progreso_porcentaje = COALESCE(b.progreso_porcentaje, 0)
   FROM bubble_tareas b
   WHERE t.bubble_id = b._id;
   
   -- Update tarea_padre_id (solo 50 registros)
   UPDATE tareas t
   SET tarea_padre_id = (
     SELECT id FROM tareas t_padre 
     WHERE t_padre.bubble_id = b.tarea_padre_bubble_id
   )
   FROM bubble_tareas b
   WHERE t.bubble_id = b._id AND b.tarea_padre_id IS NOT NULL;
   ```

3. **Validate Migration**
   ```sql
   -- Verificar count
   SELECT COUNT(*) FROM tareas WHERE progreso_porcentaje > 0;  -- Should be ~400
   SELECT COUNT(*) FROM tareas WHERE tarea_padre_id IS NOT NULL;  -- Should be ~50
   
   -- Verificar no hay gaps
   SELECT COUNT(*) FROM tareas WHERE tarea_padre_id IS NOT NULL 
   AND NOT EXISTS (SELECT 1 FROM tareas t_padre WHERE t_padre.id = tarea_padre_id);  -- Should be 0
   ```

4. **Update RLS Policy** (si aplica)
   ```sql
   -- Verificar que RLS policy en tareas funciona con tarea_padre_id
   -- Subtareas heredan tenant_id de padre automáticamente
   ```

5. **Update Backend Queries**
   - Modificar `getTareas()` server action para incluir `progreso_porcentaje`
   - Modificar `getPlaneacion()` para incluir `tarea_padre_id` + recursive query para subtareas
   - Ejemplos en `lib/actions/tareas.ts`

6. **Update UI Components**
   - Dashboard: mostrar `progreso_porcentaje` en tarjeta de tarea
   - Planificación: expandir/contraer subtareas basado en `tarea_padre_id`
   - Reporte: incluir progreso_porcentaje en análisis

**Severidad:** 🟡 MEDIA — Queries pueden fallar en producción

---

#### ACCEPTANCE CRITERIA

- [ ] Identificar todos los orphaned records
- [ ] Decidir: ¿borrar o reparar cada uno?
- [ ] Crear reporte "FK Integrity v3.11"

---

#### TASKS

1. **Audit Queries** (ejecutar todas)
   ```sql
   -- cotizaciones.cliente_id → terceros.id
   SELECT COUNT(*) FROM cotizaciones c
   WHERE NOT EXISTS (SELECT 1 FROM terceros t WHERE t.id = c.cliente_id);
   
   -- cotizaciones_detalle.cotizacion_id → cotizaciones.id
   SELECT COUNT(*) FROM cotizaciones_detalle cd
   WHERE NOT EXISTS (SELECT 1 FROM cotizaciones c WHERE c.id = cd.cotizacion_id);
   
   -- tareas.cotizacion_id → cotizaciones.id (NULL es OK)
   SELECT COUNT(*) FROM tareas t
   WHERE t.cotizacion_id IS NOT NULL
   AND NOT EXISTS (SELECT 1 FROM cotizaciones c WHERE c.id = t.cotizacion_id);
   
   -- tareas.cliente_id → terceros.id
   SELECT COUNT(*) FROM tareas t
   WHERE NOT EXISTS (SELECT 1 FROM terceros t2 WHERE t2.id = t.cliente_id);
   
   -- tareas_fechas.tarea_id → tareas.id
   SELECT COUNT(*) FROM tareas_fechas tf
   WHERE NOT EXISTS (SELECT 1 FROM tareas t WHERE t.id = tf.tarea_id);
   
   -- tareas_recursos.tarea_id → tareas.id
   SELECT COUNT(*) FROM tareas_recursos tr
   WHERE NOT EXISTS (SELECT 1 FROM tareas t WHERE t.id = tr.tarea_id);
   
   -- tareas_recursos.maquinaria_id → maquinarias.id
   SELECT COUNT(*) FROM tareas_recursos tr
   WHERE tr.maquinaria_id IS NOT NULL
   AND NOT EXISTS (SELECT 1 FROM maquinarias m WHERE m.id = tr.maquinaria_id);
   
   -- tareas_recursos.personal_id → profiles.id
   SELECT COUNT(*) FROM tareas_recursos tr
   WHERE tr.personal_id IS NOT NULL
   AND NOT EXISTS (SELECT 1 FROM profiles p WHERE p.id = tr.personal_id);
   
   -- reportes_maquinaria.maquinaria_id → maquinarias.id
   SELECT COUNT(*) FROM reportes_maquinaria rm
   WHERE NOT EXISTS (SELECT 1 FROM maquinarias m WHERE m.id = rm.maquinaria_id);
   
   -- inspecciones.maquinaria_id → maquinarias.id
   SELECT COUNT(*) FROM inspecciones i
   WHERE NOT EXISTS (SELECT 1 FROM maquinarias m WHERE m.id = i.maquinaria_id);
   ```

2. **If Orphans Found → Remediate**
   ```sql
   -- Option A: Delete orphans
   DELETE FROM cotizaciones_detalle cd
   WHERE NOT EXISTS (SELECT 1 FROM cotizaciones c WHERE c.id = cd.cotizacion_id);
   
   -- Option B: Repair (si FK tiene valor conocido)
   UPDATE tareas SET cliente_id = <known_default>
   WHERE NOT EXISTS (SELECT 1 FROM terceros WHERE id = cliente_id);
   ```

3. **Create Report**
   - Document orphaned records found

## TICKET 2: REP-3.11-002 — tareas_recursos Validación ⚠️

### Auditoría Completada — BLOCKER IDENTIFICADO

**Estadísticas:**
```
Total tareas activas:     1,000
Con recursos asignados:     135 (13.50%)
Sin recursos:               865 (86.50%) ← CRÍTICO
FK Integrity:            ✅ OK (0 orphans)
Maquinarias references:      44 (OK)
Profiles references:         71 (OK)
```

**Root Cause:** La migración de `tareas_recursos` desde Bubble fue **incompleta**. Solo 135 de 1,000 tareas tienen asignaciones de maquinaria/personal en la junction table.

**Impacto:**
- ⚠️ Planificación visual (`/planificacion`) no funciona correctamente
- ⚠️ Timeline de recursos no disponible
- ⚠️ Falta de datos para reportes de carga de equipos

**Aceptación Criteria:**
- ❌ 95%+ tareas con recursos
- ❌ FK integrity: 0 orphans (✅ OK, pero bajo coverage)
- ❌ Reporte documentado
- ❌ `/planificacion` UI validated

**Conclusión:** ❌ **RECHAZADO** — Requiere remediation.

### Plan de Acción Requerido

**Paso 1: Auditar Bubble (30 min)**
```sql
-- En Bubble API: Contar tareas_recursos por tenant
SELECT COUNT(*) FROM bubble_tareas_recursos
WHERE tarea_id NOT IN (SELECT DISTINCT tarea_id FROM supabase.tareas_recursos)
  AND tenant IN ('CISE', 'GRUAS')
```

**Paso 2: Migrar (4-5 horas)**
```bash
# Crear script de migración de datos desde Bubble
npx tsx scripts/migrate-tareas-recursos-from-bubble.ts --run
```

**Paso 3: Validar (30 min)**
- Ejecutar auditoría nuevamente
- Confirmar cobertura ≥ 95%
- Test manual en `/planificacion`

**Paso 4: Documentación**
- Actualizar `docs/AUDITORÍA_tareas_recursos_v3.11.md`
- Registrar datos migrados
